# Get Task Status

Query the execution status of 3D generation tasks and retrieve generated model files when complete.

## Endpoint

```http
GET https://api.poyo.ai/api/generate/status/{task_id}
```

## Authentication

All requests require Bearer token authentication:

```bash
Authorization: Bearer YOUR_API_KEY
```

## Parameters

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `task_id` | string | Yes | Unique task identifier returned from the submit endpoint |

## Response

```json
{
  "code": 200,
  "data": {
    "task_id": "9OTM0HH9LBS63LF0",
    "status": "finished",
    "files": [
      {
        "file_url": "https://cdn.doculator.org/3d/9OTM0HH9LBS63LF0/9OTM0HH9LBS63LF0_1.glb",
        "file_type": "3d"
      },
      {
        "file_url": "https://cdn.doculator.org/3d/9OTM0HH9LBS63LF0/9OTM0HH9LBS63LF0_2.png",
        "file_type": "image"
      },
      {
        "file_url": "https://cdn.doculator.org/3d/9OTM0HH9LBS63LF0/9OTM0HH9LBS63LF0_3.glb",
        "file_type": "3d"
      }

    "created_time": "2026-05-12T02:36:12",
    "progress": 100
  }

```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `code` | integer | HTTP status code (200 for success) |
| `data.task_id` | string | Unique task identifier |
| `data.status` | string | Task status: `not_started`, `running`, `finished`, `failed` |
| `data.files` | array | Generated 3D model, preview image, and supporting files when status is `finished` |
| `data.files[].label` | string | Optional file label, such as `model_glb`, `thumbnail`, a model format label, texture label, or animation label |
| `data.files[].file_url` | string | Direct URL to generated file |
| `data.files[].file_type` | string | Type of generated file: `3d`, `image`, or `other` |
| `data.files[].format` | string | Optional file format, such as `glb`, `fbx`, `obj`, `usdz`, `stl`, or `png` |
| `data.files[].content_type` | string | Optional MIME type of the generated file |
| `data.files[].file_name` | string | Optional file name |
| `data.files[].file_size` | integer | Optional file size in bytes |
| `data.created_time` | string | ISO 8601 timestamp of task creation |
| `data.progress` | number | Completion percentage (0-100) |
| `data.error_message` | string | Error description when status is `failed`; omitted for successful tasks |

### Status Values

| Status | Description |
|--------|-------------|
| `not_started` | Task is queued, waiting to be processed |
| `running` | Task is currently being generated |
| `finished` | Task completed successfully, results available in `files` |
| `failed` | Task failed, error details in `error_message` |

## Code Example

```bash
curl -X GET https://api.poyo.ai/api/generate/status/task-unified-1757165031-uyujaw3d \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Important Notes

<Warning>
  **Content Validity**: Generated 3D files are accessible for **24 hours** after creation. Download and save your content promptly.
</Warning>

<Tip>
  **Webhooks**: Instead of polling, you can provide a `callback_url` when submitting tasks to receive automatic notifications. See [3D Webhook Callbacks](/api-manual/task-management/3d-webhooks).
</Tip>
