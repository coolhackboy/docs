# Happy Horse

`happy-horse` supports both text-to-video and image-to-video generation through one public model ID.

## Available Model

- **happy-horse** - Alibaba Happy Horse 1.0 text-to-video and image-to-video

## Workflows

### Text to Video

Send a `prompt` without `image_urls`.

### Image to Video

Send a single-item `image_urls` array. The image is used as the first frame. `prompt` is optional in this workflow.

## Required Parameters

- **model**: `happy-horse`
- **input.prompt**: Required for text-to-video. Optional for image-to-video.

## Optional Parameters

- **image_urls**: Single-item first-frame image URL array
- **aspect_ratio**: Text-to-video only. `16:9`, `9:16`, `1:1`, `4:3`, or `3:4`. Default is `16:9`
- **resolution**: `720p` or `1080p`. Default is `1080p`
- **duration**: Integer seconds from `3` to `15`. Default is `5`
- **enable_safety_checker**: Optional boolean

## Pricing

- **720p**: 16 credits per second
- **1080p**: 32 credits per second

## Notes

- `image_urls` must resolve to exactly one image.
- For text-to-video, omit image fields and use `aspect_ratio`.
