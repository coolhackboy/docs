## Usage Guide

- This endpoint checks the current status of an asset created through [Create Asset](/api-manual/file-series/create-asset)
- When the upstream transfer succeeds, the response returns the final asset `url`
- This endpoint is useful when you need to confirm whether a transferred asset is ready before using it in downstream workflows

## Typical Status Values

- `Processing`: The asset is still being processed upstream
- `Active`: The asset is ready and the final `url` is available
- `Failed`: The transfer failed and `error_message` may contain the upstream reason
- `unknown`: Fallback status used when the upstream service does not return a standard state field

## Developer Notes

- Keep the `asset_id` returned by [Create Asset](/api-manual/file-series/create-asset)
- Poll this endpoint until the status becomes `Active` or `Failed`
- When the status is `Active`, persist the returned `url` if your workflow needs a stable provider-hosted asset reference
