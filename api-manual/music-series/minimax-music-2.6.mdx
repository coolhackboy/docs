<Tip>
1. After submission, a `task_id` will be returned. If you provided a `callback_url`, when the task status becomes `finished` or `failed`, a POST request will be sent to the `callback_url`.
2. Regardless of whether `callback_url` is provided, you can retrieve the response result through the unified [Query Music Detail](/api-manual/music-series/query-music-detail) endpoint.
</Tip>

## Usage Guide

- This endpoint generates a music track from a style prompt
- You can provide custom lyrics for vocal generation or enable instrumental mode for music without vocals
- Final results are returned asynchronously through `callback_url` or [Query Music Detail](/api-manual/music-series/query-music-detail)

## Parameter Details

- `prompt` (required): Music prompt describing style, mood, genre, instrumentation, tempo, or scene
  - Length: `10-2000` characters
  - Be specific about the musical direction you want in the final track

- `lyrics`: Optional lyrics content for vocal generation
  - Required unless `is_instrumental` is `true`
  - Maximum length: `3500` characters

- `is_instrumental`: Optional boolean
  - Set to `true` to generate instrumental-only music without vocals
  - When `true`, omit `lyrics`

- `audio_setting`: Optional output audio settings object
  - `sample_rate`: `16000`, `24000`, `32000`, `44100`
  - `bitrate`: `32000`, `64000`, `128000`, `256000`
  - `format`: `mp3`, `wav`, `pcm`
  - Only these three keys are supported

## Developer Notes

- If `is_instrumental` is `false`, provide `lyrics` to avoid validation errors
- Query results for this model return `audio_id`, `audio_url`, `title`, `duration`, and `prompt`
